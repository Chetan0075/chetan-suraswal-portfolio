The Problem

Delays Were Reported,
Never Predicted.

In project-driven environments, delay usually gets noticed the same week a deadline is missed — by then it's too late to fix. Resource assignment was often based on gut feel and seniority rather than actual workload risk, and there was no early signal for which tasks were quietly heading toward failure.

The Dataset

Dataset Onename placeholder

A task-and-resource-level operations dataset — each record covers a task with its assigned resource, estimated vs. actual effort, deadline, completion timestamp, and resource experience level. (Swap in the real dataset name/source here.)

Data Quality & Assumptions

  • Effort estimates assumed reasonably accurate at task creation time
  • Resource experience treated as a static attribute, not time-varying
  • External blockers (e.g. client delays) not separated from internal delay causes

The Approach

Six Stages,
Raw Tasks to Delay Risk.

01

Data Sources

Pulled task, resource, and project metadata from Dataset One — assignment history, effort estimates, and delivery timestamps.

02

SQL Warehouse

Structured the raw records into a queryable warehouse — joining task, resource, and project tables, and standardizing date and effort fields for analysis.

03

Feature Engineering

Engineered features like estimated-vs-actual effort ratio, resource experience level, task criticality, and rolling delay rate per resource.

04

Analysis

Tested which features actually predict delay. Effort ratio and reassignment timing mattered far more than raw seniority or experience.

05

Power BI

Built a live dashboard surfacing delay-risk by task, resource, and time period — so a manager can catch risk before a deadline, not after.

06

Insights

Turned the model output into specific actions: which tasks to flag, which resources to reassign, and when to rebalance workload.

What It Found

Insights & Decisions.

Key Business Insights

  • Experience alone does not guarantee faster delivery
  • 18% of tasks contribute to over 40% of total delay
  • Resource reassignment timing heavily impacts outcomes

Decisions Enabled

  • Prioritized high-risk tasks for senior resources
  • Flagged tasks exceeding 120% estimated effort
  • Rebalanced workload during peak delivery weeks
  • Delayed non-critical tasks to protect deadlines
